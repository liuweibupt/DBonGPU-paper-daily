1 Debunking the Myth of Join Ordering: Toward Robust SQL Analytics
SIGMOD 2025 (Proc. ACM Manag. Data, Vol. 3, No. 3) 2025
Junyi Zhao, Kai Su, Yifei Yang, Xiangyao Yu, Paraschos Koutris 等 (6 位作者)
📝 中文摘要
连接顺序优化对于实现良好的查询性能至关重要。尽管经过了几十年的研究和实践，现代查询优化器仍然可能生成比最优计划慢几个数量级的次优连接计划。现有的鲁棒查询处理研究通常缺乏对连接顺序鲁棒性的理论保证，同时牺牲了查询性能。在本文中，我们从鲁棒性的角度重新审视了近期提出的谓词迁移（Predicate Transfer）技术。我们引入了两种新算法 LargestRoot 和 SafeSubjoin，并提出了鲁棒谓词迁移（RPT），该技术在理论上可以证明对无环查询的任意连接顺序具有鲁棒性。我们将 RPT 集成到 DuckDB 中，并在 TPC-H、JOB 和 TPC-DS 基准测试的全部查询上进行了评估。实验结果表明，RPT 将连接顺序鲁棒性提升了数个数量级。使用 RPT 后，对于单个无环查询，随机连接顺序中最大与最小执行时间之比仅为 1.6 倍（大多数评估查询的比值接近 1）。同时，应用 RPT 还将端到端查询性能提升了 1.5 倍（每查询几何均值）。

📝 中文摘要
GPU 由于提供高带宽和大规模并行性，越来越多地被用于数据库分析。然而，对于 GPU 在何时以及为何优于 CPU 执行数据库工作负载，理解仍然有限。本文采用基于模型的方法来理解在 GPU 与 CPU 上运行查询的性能增益何时以及为何偏离带宽比（现代硬件上约为 16 倍）。我们研究了基本数据库算子（扫描、过滤、连接和聚合）在 CPU 和 GPU 上的性能。我们开发了一个简单但有效的性能模型，考虑了计算、内存和通信成本来解释观察到的性能差异。我们的模型表明，对于许多算子，加速比不是受限于计算而是受限于内存或通信瓶颈，并且加速比在不同算子和数据特征之间差异显著。我们还提供了 GPU 加速对数据库分析有益的指导准则。

GPU 对数据库的加速远不是简单的带宽比——实际加速取决于算子类型和数据特征，许多场景下瓶颈在内存或通信而非计算；理解这一模型才能正确选择何时使用 GPU。

1. 首次系统性地建模 GPU vs CPU 数据库算子性能差异；2. 开发考虑计算、内存和通信成本的简单有效性能模型；3. 揭示加速比受限于内存/通信瓶颈而非计算的真相；4. 提供 GPU 加速数据库分析的实际指导准则。

12 Rethinking Analytical Processing in the GPU Era
CIDR 2026 / arXiv 2025 2025
Bobbi Yogatama, Yifei Yang, Kevin Kristensen, Devesh Sarda, Abigale Kim 等 (12 位作者)
📝 中文摘要
GPU 驱动的数据分析时代已经到来。在本文中，我们认为硬件（如更大的 GPU 内存、更快的互连和 IO，以及不断下降的成本）和软件（如可组合的数据系统和成熟的库）方面的最新进展，已经消除了限制 GPU 数据分析广泛采用的关键障碍。我们推出了 Sirius，一个开源的 GPU 原生 SQL 引擎原型，可为多种数据系统提供即插即用的加速。Sirius 将 GPU 视为主要引擎，利用 libcudf 等库实现高性能关系算子。它通过利用标准 Substrait 查询表示，在不改变用户界面的情况下替换 CPU 引擎，为现有数据库提供即插即用的加速。Sirius 在与单机 DuckDB 集成时，在 TPC-H 和 ClickBench 上分别实现了 8.3 倍和 7.4 倍更好的成本效率；与 Apache Doris 分布式引擎集成时，速度提升高达 12.5 倍。
